Manual directed tests cannot cover the state space of multi-protocol packet
processing — the cross of payload shape, header validity, and back-pressure
conditions explodes past anything you can author by hand. The methodology needed to
be reusable across three tape-out cycles.

## Architecture

A layered UVM environment with protocol-aware agents for AXI4 and a custom streaming
bus. A reference-model scoreboard checks every transaction; assertion-based protocol
checks live at every interface boundary.

Seed management is automated: a Python harness records the seed of each regression
failure and re-runs the failing seed under Verdi for waveform capture without
engineer intervention.

## Implementation

The constrained-random AXI4 burst sequence is the workhorse of the regression. It
bounds burst length, fixes burst size, and aligns the address — the alignment check
would otherwise dominate failure noise.

```system-verilog
// Sequence: random AXI4 burst with bounded length
class axi_burst_seq extends uvm_sequence #(axi_item);
  `uvm_object_utils(axi_burst_seq)

  task body();
    axi_item item;
    repeat (cfg.num_bursts) begin
      item = axi_item::type_id::create("item");
      start_item(item);
      assert(item.randomize() with {
        burst_len inside {[4:64]};
        burst_size == 3;
        addr % (1 << burst_size) == 0;
      });
      finish_item(item);
    end
  endtask
endclass
```
