Educational RISC-V cores routinely cut corners on hazard resolution and branch
prediction. The objective: bridge an academic ISA implementation with the pipeline
discipline of a production design — at FPGA scale.

## Architecture

A classic five-stage IF/ID/EX/MEM/WB pipeline with full forwarding and stall logic.
A 1024-entry gshare BHT handles branch prediction; the global history register is
updated speculatively and restored on misprediction.

A custom V-extension subset adds SIMD on 128-bit vectors with chained MAC, sufficient
for hand-written inner kernels of small CNN inference.

## Implementation

Forwarding is the pivot point of the pipeline. The unit shown below resolves EX→EX
and MEM→EX hazards in a single cycle.

```system-verilog
// Forwarding unit — resolves RAW hazards across EX/MEM/WB
module forwarding (
  input  logic [4:0] rs1_id, rs2_id,
  input  logic [4:0] rd_ex_mem, rd_mem_wb,
  input  logic       reg_we_ex_mem, reg_we_mem_wb,
  output logic [1:0] fwd_a, fwd_b
);
  always_comb begin
    fwd_a = 2'b00; fwd_b = 2'b00;
    if (reg_we_ex_mem && rd_ex_mem != 0 && rd_ex_mem == rs1_id) fwd_a = 2'b10;
    else if (reg_we_mem_wb && rd_mem_wb != 0 && rd_mem_wb == rs1_id) fwd_a = 2'b01;
    if (reg_we_ex_mem && rd_ex_mem != 0 && rd_ex_mem == rs2_id) fwd_b = 2'b10;
    else if (reg_we_mem_wb && rd_mem_wb != 0 && rd_mem_wb == rs2_id) fwd_b = 2'b01;
  end
endmodule
```

## Verification

Coverage-driven UVM functional coverage targets every RV32I instruction class plus
all hazard scenarios (load-use, EX→EX forwarding, MEM→EX forwarding,
branch-on-forwarded). HW/SW co-simulation against the official compliance suite —
`riscv-tests` and `riscof` — reaches 100% pass.
