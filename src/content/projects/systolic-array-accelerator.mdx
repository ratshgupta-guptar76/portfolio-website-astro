Dense matrix multiplication dominates inference latency at the edge. General-purpose
datapaths waste area on logic that never executes, while fixed accelerators sacrifice
generality. The objective: a small, dense compute fabric that runs INT8 GEMM near the
arithmetic limits of the process.

## Architecture

Weight-stationary 8×8 PE grid with output-stationary accumulation. Configurable
INT8/INT16 datapath. Double-buffered SRAM banks streamed through a custom AXI4 DMA
controller. Layer-aware activation memory layout avoids bank conflicts under the
worst-case mapping.

The arithmetic intensity of the inner loop is

$$
\mathrm{AI} = \frac{2 \cdot N^3}{N^2 \cdot (W_a + W_w + W_o)}
$$

so the goal of the memory subsystem is to keep $\mathrm{AI}$ bandwidth-bound at the
SRAM, not at the off-chip interface.

## Implementation

The processing element registers the weight on load and forwards the activation
combinationally. Registering the multiply (not the accumulate) is what keeps the
critical path inside a single MAC.

```system-verilog
// Processing element — weight-stationary MAC
module pe #(
  parameter int DW = 8
) (
  input  logic              clk,
  input  logic              rst_n,
  input  logic              we_n,
  input  logic signed [DW-1:0] act_in,
  input  logic signed [DW-1:0] w_in,
  input  logic signed [31:0]   psum_in,
  output logic signed [DW-1:0] act_out,
  output logic signed [31:0]   psum_out
);
  logic signed [DW-1:0] w_q;

  always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n)         w_q <= '0;
    else if (!we_n)     w_q <= w_in;
  end

  assign act_out  = act_in;
  assign psum_out = psum_in + (act_in * w_q);
endmodule
```
